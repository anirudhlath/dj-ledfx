Spatial Mapping & Compositor

How 1D effect strips get projected onto 3D device positions

SpatialMapping Protocol

Stateless protocol — maps 3D positions to 0.0–1.0 strip indices
class SpatialMapping(Protocol):
def map_positions(
self,
positions: NDArray[np.float64], # (N, 3) world coords
time: float, # for animated mappings (rotation, etc.)
) -> NDArray[np.float64]: # (N,) values in [0.0, 1.0]
...

Built-in Mappings

LinearMapping
Projects positions onto a direction vector
dot(pos, direction) → normalized to [0, 1]
Params: direction, origin (optional)
RadialMapping
Distance from a center point
dist(pos, center) → normalized to [0, 1]
Params: center, max_radius (optional, auto from scene)

Compositor Pipeline

SpatialCompositor — per-device color sampling
class SpatialCompositor:
# Initialized once with scene data
scene: SceneModel
mapping: SpatialMapping
# Pre-computed at init:
_led_positions: dict[str, NDArray] # device_id → (N, 3)
_strip_indices: dict[str, NDArray] # device_id → (N,) cached mapping

def composite(
self,
effect_strip: NDArray[np.uint8], # (led_count, 3) from Effect.render()
device_id: str,
time: float,
) -> NDArray[np.uint8]: # (device_led_count, 3)
# 1. Get cached strip indices for this device (or recompute if animated)
# 2. Sample effect_strip at those indices (numpy fancy indexing)
# 3. Return per-LED colors for this device

Sampling: indices map to strip positions via:
# strip_idx = mapping.map_positions(led_positions, time) → [0.0, 1.0]
# pixel_idx = (strip_idx * (strip_len - 1)).astype(int)
# colors = effect_strip[pixel_idx] → (N, 3)

Integration with Scheduler

Where compositing happens in the per-device send loop
1
Per-device send loop picks frame from ring buffer (unchanged)
2
NEW: compositor.composite(frame.colors, device_id, time)
3
Send per-device colors to adapter (unchanged)
Compositing is a single numpy indexing operation — sub-microsecond for typical device sizes.
No spatial config? Compositor is None, send loop falls back to current broadcast behavior.
