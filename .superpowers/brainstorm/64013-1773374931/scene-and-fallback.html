Adapter Geometry Reporting

Adapters optionally report their physical geometry
# New optional property on DeviceAdapter ABC
@property
def geometry(self) -> DeviceGeometry | None:
return None # default: adapter doesn't know its geometry

Adapter implementations:
OpenRGBAdapter
→ returns None (geometry from config only)
LIFXBulbAdapter
→ returns PointGeometry()
LIFXStripAdapter
→ returns StripGeometry(led_count, ...)
LIFXTileChainAdapter
→ returns MatrixGeometry(tiles=...) from discovery

Resolution order for geometry:
1. TOML config explicit geometry → wins
2. Adapter-reported geometry → auto-discovered
3. Fallback → PointGeometry() if device has 1 LED, StripGeometry(direction=[1,0,0]) otherwise

Fallback Behavior

Graceful degradation — spatial mapping is always optional
FULL SCENE
[scene] defined with device placements
→ Compositor active, per-LED spatial sampling
PARTIAL
[scene] exists but some devices missing from config
→ Unmapped devices get broadcast behavior (current default), log WARNING
NO SCENE
No [scene] in config
→ Compositor is None, pipeline unchanged from MVP behavior
Zero breaking changes. Existing configs with no [scene] section work exactly as before.
